Native ZDX Framework Integration for Multicharacter
The current zdx_multicharacter was essentially a modified multicharacter script built for esx and qb. Since you are running your own custom ZDX Framework (zdx_core), the script falls back to detecting ESX compatibility mode and attempts to read from the users database table instead of your actual zdx_users table.

This plan outlines the steps to natively integrate zdx_multicharacter with zdx_core.
